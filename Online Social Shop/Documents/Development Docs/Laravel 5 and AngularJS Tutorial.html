For this first Laravel 5 and AngularJS tutorial we are going to build a simple Todo Application. We will use the resource routing of Laravel 5 to crete a REST API for the backend of the application and the powerfull of AngularJS for the frontend.

This tutorial suppose you have already installed Laravel and all the LAMP stack on your development machine. If not, i can suggest you to follow the official Laravel guide to get Vagrant Homestead installed.

If you want to read another tutorial about Laravel 5 and AngularJS mixed in a different way you can read the RELOADED TUTORIAL.

Laravel 5 REST API

Let’s create the application as a normal Laravel app.

$laravel new todoApp

First thing we can do is create the database table for our todo. Laravel migration come in help.

$php artisan make:migration create_todo_table --table=todos

In the database/migrate folder of your app it will create an empty migration file.
Open this file and add some lines to describe our todo model.

class CreateTodoTable extends Migration {

Schema::drop('todos');

To take this tutorial simple we add just 2 fields to describe the Todo (title of type string and done of type boolean).

<?php namespace App\Http\Controllers;
 
use App\Http\Controllers\Controller;
use App\Todo;
use Request;
 
class TodosController extends Controller {
 
/**
* Display a listing of the resource.
*
* @return Response
*/
public function index() {
 
$todos = Todo::all();
return $todos;
}
 
/**
* Store a newly created resource in storage.
*
* @return Response
*/
public function store() {
$todo = Todo::create(Request::all());
return $todo;
}
 
/**
* Update the specified resource in storage.
*
* @param  int  $id
* @return Response
*/
public function update($id) {
$todo = Todo::find($id);
$todo->done = Request::input('done');
$todo->save();
 
return $todo;
}
 
/**
* Remove the specified resource from storage.
*
* @param  int  $id
* @return Response
*/
public function destroy($id) {
Todo::destroy($id);
}
 
}
 
 

At this point our REST API backend is done and ready to serve our application. You can test it with a REST client like POSTMAN in Chrome.

To be sure all is implemented in the right manner we can list the routes enabled for our app.

$ php artisan route:list

AngularJS frontend

Mixing this two framework is easy but there is some consideration before starting. The Blade engine of Laravel use the same tag “{{ }}” to put variable in the view. It’s the same that’s AngulaJS use.

There is some various way to integrate the two framework. We will take the one i think is the simplest and also the most versatile way, leaving Laravel template as it is and integrate a angular application when it’s needed.

Also for the AngularJS app files i’ve putted it in directly in the public folder of the Laravel application. Teorically you can use the gulp engine coming with Laravel to minimize the and mix the js code, but it’s far from the purpose of this tutorial.

Let’s add AngularJS in our view master template and also the link to our app.js. In the app.blade.php add this lines :

var app = angular.module('todoApp', [], function($interpolateProvider) {
$interpolateProvider.startSymbol('<%');
$interpolateProvider.endSymbol('%>');
});
 
app.controller('todoController', function($scope, $http) {
 
$scope.todos = [];
$scope.loading = false;
 
$scope.init = function() {
$scope.loading = true;
$http.get('/api/todos').
success(function(data, status, headers, config) {
$scope.todos = data;
$scope.loading = false;
 
});
}
 
$scope.addTodo = function() {
$scope.loading = true;
 
$http.post('/api/todos', {
title: $scope.todo.title,
done: $scope.todo.done
}).success(function(data, status, headers, config) {
$scope.todos.push(data);
$scope.todo = '';
$scope.loading = false;
 
});
};
 
$scope.updateTodo = function(todo) {
$scope.loading = true;
 
$http.put('/api/todos/' + todo.id, {
title: todo.title,
done: todo.done
}).success(function(data, status, headers, config) {
todo = data;
$scope.loading = false;
 
});;
};
 
$scope.deleteTodo = function(index) {
$scope.loading = true;
 
var todo = $scope.todos[index];
 
$http.delete('/api/todos/' + todo.id)
.success(function() {
$scope.todos.splice(index, 1);
$scope.loading = false;
 
});;
};
 
$scope.init();
 
});
 
 

First thing to notice is the $interpolateProvider used to change the bracket symbol used to put variable in the view.

1

$interpolateProvider.startSymbol('<%');
$interpolateProvider.endSymbol('%>');
 
 

Then, for each method we use the $http service to invoke REST api of the backend.

There is also a $scope.loading variable we use to show or hide the loading spinning icon. All the function are very easy to understand, after each invocation a .success function to update the interface.

Just a side note on the delete, here we use the index of the array passed as parameter on the URL to find  the Todo Objects in the array and call the delete method on it id.

Conclusion

This is the frist approach to Laravel 5 mixed with Angular. My idea is to take the power of both frame to have a mixed application, made for some part of little single page apps and the entire structure of the site made as a classic php.
